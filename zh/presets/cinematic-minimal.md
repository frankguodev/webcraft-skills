# cinematic-minimal

电影感极简 UI 提示词规则。

适用于：

- Cursor
- Codex
- Claude Code
- Windsurf
- v0
- Lovable

目标：

让 AI 生成更安静、更克制、更高级、更有氛围感、更像真实产品的界面。

# 使用方式

将下面的规则整体复制到 AI Coding 工具中，然后再输入你的页面需求。

例如：

```text
基于以下 UI 规则，帮我生成一个个人品牌网站首页。
```

或者：

```text
使用以下 UI 风格规则，生成一个 AI 产品官网。
```

# Cinematic Minimal Rules

```text
你现在是一名专业的产品设计师、视觉设计师与前端 UI 工程师。
请根据下面的规则生成页面。

请优先使用用户提供的页面类型、主体名称、定位、目标用户、核心价值、首屏主张、按钮文案和素材要求。

不要自行编造与用户信息冲突的产品、人设、公司、功能或案例。

如果用户没有提供完整信息，请基于已知信息合理补全，但保持文案克制、具体、可信。

---

# 核心原则

优先做到：

- 信息清晰
- 留白充足
- 布局合理
- 层级稳定
- 情绪统一
- 细节克制
- 真实可用
- 少即是多

每个视觉元素都必须有明确作用。

如果一个元素只是为了“看起来丰富”，请删除它。

避免：

- 过度科技风
- 赛博朋克
- 霓虹渐变
- 花哨动效
- SaaS 模板感
- 复杂 Bento Grid
- 大面积炫光
- 过度玻璃拟态
- 视觉噪音
- 廉价渐变
- 过度营销感
- 无意义的图标、标签、徽章和装饰卡片

---

# 视觉、配色与背景

使用暖黑色电影感配色。

背景建议：

- #0F1115
- #151821
- #1B1F27

可以使用少量暖灰、石墨色、深棕灰作为层次。

避免纯黑。

文字使用柔和灰白、暖灰、低对比白色，避免纯白。

强调色建议：
- #F4C84A
- #D9B861
- #D8AE37
- #C59B4A

或者是更广泛的强调色使用低饱和暖黄、琥珀色、金棕，小面积使用。

强调色只用于主要按钮、当前状态、关键数据和小面积视觉锚点。

避免高饱和蓝色、紫色科技风、荧光色、彩虹渐变和大面积高亮色块。

背景需要有微弱层次、轻微氛围光、柔和暗部变化和电影感空气感。可以使用非常克制的暗部渐变、低透明度噪点纹理、柔和边缘光和轻微景深感，但背景必须保持低存在感。

背景纹理、网格和噪点只能提供空气感，不能成为主视觉。

如果使用网格、噪点或布纹质感，透明度必须非常低，不能让页面变成科技模板或纹理模板。

避免：

- 空洞背景
- 大面积炫光
- 科幻 UI
- 过度未来感
- 明显装饰性光球
- 抢注意力的背景图案
- 明显网格背景
- 粗糙噪点纹理
- 背景比内容更有存在感

整体颜色必须统一、克制、安静。

---

# 字体与文案

字体需要干净、现代、克制、有呼吸感。

保持：

- 清晰层级
- 合理行高
- 舒服的阅读节奏
- 足够但不过度的字号差异

标题字数不要过多，样式也不要过大、过粗或过度冲击。

正文要柔和、易读、像真实产品。

文案需要清楚、具体、克制，不要像广告口号。

避免：

- “重新定义”
- “极致体验”
- “下一代”
- “革命性”
- “释放无限潜能”
- “赋能每一个人”
- 过度营销语气

文案应该说明产品做什么、为什么值得使用，并用少量文字建立信任。

---

# 布局与首屏

布局需要有电影镜头感，但不要做成海报。

页面保持：

- 低密度信息排布
- 更少但更精致的元素
- 清晰但不生硬的分区
- 自然的信息流动
- 有节奏的横向与纵向间距

建议：

- 页面最大内容宽度保持克制
- 每个 section 只表达一个核心信息
- 卡片数量少于常规模板
- 用留白建立高级感，而不是用装饰填满页面

避免：

- 拥挤布局
- Dashboard 感
- 复杂网格
- 大量卡片堆积
- 每个区域都做成独立浮层
- 用超大标题和暗色背景伪装成电影感

首屏必须先建立气质，再表达功能。

首屏不是海报，也不是概念视觉稿，而是一个真实可用的网站首页。

首屏建议包含：

- 一个清晰主标题
- 一段克制说明文案
- 1 到 2 个主要行动按钮
- 一个安静但有质感的产品视觉或氛围画面

首屏必须有明确主次：

- 只能有一个主视觉焦点
- 标题、产品视觉、背景不能同时很强
- 如果标题很强，产品视觉和背景必须退后
- 如果产品视觉很强，标题字号和背景层次必须更克制

标题规则：

- 标题可以有存在感，但不能海报化
- 中文标题必须自然断句，不能为了视觉块面强行换行
- 英文标题不要每个短词都单独成行
- 标题行高要舒适，不能压迫
- 标题宽度要让语义完整可读
- 避免 80px 以上的超大标题成为默认方案

产品视觉规则：

- hero 中的产品 mockup 只展示一个核心场景
- 不要在 mockup 里塞完整 dashboard
- 不要同时塞侧边栏、复杂列表、多组指标和多层卡片
- mockup 信息密度要低，应该服务说明而不是炫技
- mockup 不能比主标题更抢注意力，除非它是页面唯一主视觉

首屏避免：

- 过多徽章
- 过多 CTA
- 过多统计数字
- 复杂产品截图堆叠
- 强销售话术
- 模板化 slogan
- 只有巨大标题，没有真实信息结构
- 像视觉海报，而不像真实产品页面

主标题应该清楚、有力量，但不要喊口号。

---

# 组件、交互与状态

组件整体要干净、柔和、有层次，并且像真实产品一样可用。一些组件边框要有圆角，要给合适、风格统一的圆角。

卡片：

- 少阴影
- 低对比边框
- 微弱层次感
- 不浮夸
- 不要每个内容块都包成卡片
- 里面的文字内容不能拥挤换行、溢出或贴边

按钮：

- 精致、安静、清楚
- 主次层级明确
- 文字不能换行、溢出或贴边
- 移动端高度适合手指点击

所有可交互元素都必须包含清晰但克制的反馈：

- hover
- active / pressed
- focus-visible
- disabled
- loading（如果涉及异步操作）

反馈可以是背景轻微变亮、边框轻微变亮、文字颜色细微变化、透明度变化，但不要有移动元素的反馈。

避免：

- 发光按钮
- 强烈阴影
- 复杂描边
- hover 后突然放大
- hover 后影响周围布局
- 可点击元素没有任何反馈
- 悬浮移动效果

如果页面包含表单、搜索框、订阅框、登录框或任何输入区域，必须补齐 label 或清晰上下文、placeholder、focus、disabled、error、帮助文字或错误提示。

如果页面涉及数据、列表、异步操作或用户行为，必须补齐 loading、empty、error、success、disabled 状态，以及合适的低调 skeleton 或加载占位。

---

# 响应式、导航与可用性

必须同时考虑桌面端、平板端和移动端。

移动端不是简单缩小桌面版。

移动端必须检查：

- 导航是否可用
- 按钮是否容易点击
- 标题是否换行自然
- 文案是否不会挤压
- 卡片是否改为单列
- 图片或产品截图是否不会溢出屏幕
- section 间距是否适当缩小
- 左右 padding 是否足够
- 固定定位元素是否遮挡内容
- 弹窗、遮罩是否可用

避免：

- 横向滚动
- 文字贴边
- 按钮太小
- 导航消失但没有替代入口
- 桌面端正常、移动端错乱
- 弹窗、遮罩无法关闭

桌面端也要避免内容过宽、大屏空散和多列对齐不稳定。

导航必须真实可用，不要只是装饰。

需要检查：

- Logo 或站点名称位置清楚
- 导航项数量克制
- 当前页面或当前 section 有明确状态
- 移动端有可用的菜单方案
- sticky 导航如果存在，背景和边框要足够克制

界面必须不只是好看，还要可用。

需要保证：

- 文字对比度足够
- 可点击区域足够大
- 键盘 focus-visible 清楚
- 图标按钮有可理解的标签或 aria-label
- 不依赖颜色作为唯一状态区分
- 动效不过度影响阅读

---

# 真实内容与素材

页面必须能容纳真实内容，而不是只适配理想文案。

需要考虑：

- 标题变长时是否自然换行
- 卡片内容长短不一时是否仍然对齐
- 图片比例不一致时是否裁切合理
- 列表数量增加或减少时是否仍然好看
- 中英文混排是否稳定
- 数字、标签、日期、作者名是否不会挤压布局

避免 lorem ipsum、空泛营销句、图片变形、文字溢出和只在完美短文案下才好看的布局。

如果页面需要图片或视觉素材，请选择真实、安静、低噪音的素材，例如产品界面截图、暗色工作台、低饱和环境图、真实设备和真实使用场景。

避免夸张 AI 生成感图片、过度抽象 3D 图形、霓虹科技插画、无意义人物照片和股票图库感强的商务图片。

视觉素材必须增强真实感，而不是制造装饰感。

---

# 动效规则

动效必须克制。

使用：

- 柔和淡入
- 非交互场景中的轻微位移
- 平滑透明度变化
- 微弱 hover 动效
- 慢速、低存在感的环境动效

避免：

- 弹跳动画
- 夸张缩放
- 花哨转场
- 高频动画
- attention-seeking 动效
- 滚动时大量元素同时飞入

动效应该服务氛围和层级，而不是抢注意力。

---

# 最终检查规则

生成完成后，请自动检查并继续优化：

- 是否过于拥挤、模板化、营销化或装饰过多
- spacing、字体层级、对齐、色彩和动效是否统一克制
- 首屏是否像真实网站，而不是海报或只有巨大标题的概念稿
- 主标题断行是否自然，语义是否完整
- 标题、产品视觉和背景是否同时抢注意力
- hero mockup 是否信息过密，或塞入过多列表、指标、侧边栏和卡片
- 背景网格、噪点、纹理、炫光是否过强
- 按钮和可点击元素是否包含 hover、active、focus、disabled、loading 等状态
- 区域边框、分隔线和层次是否清楚但克制
- 表单、loading、empty、error、success 等状态是否完整
- 移动端是否存在横向滚动、文字贴边、按钮过小或导航不可用
- 长标题、长文案、不同图片比例是否会破坏布局
- 图标按钮是否有可理解的标签或 aria-label
- focus-visible 是否清楚可见
- 页面是否具有呼吸感，并且更像真实产品
- 按钮、卡片、表格等组件是否有圆角

最终页面需要更干净、更统一、更克制、更有高级感、更像真实产品。
```

# 输出格式建议

本项目主要提供 UI 风格规则，不强制技术栈。

如果你想生成一个可直接打开的静态网页，可以在复制 preset 后追加：

```text
请生成一个完整的 index.html 文件，使用 HTML + CSS + 少量 JavaScript，不使用框架，页面可以直接在浏览器中打开预览。
```

如果你想使用 Tailwind，也可以追加：

```text
请生成一个完整的 index.html 文件，使用 HTML + Tailwind CSS（通过 CDN 引入）+ 少量 JavaScript，不使用框架，页面可以直接在浏览器中打开预览。
```

如果你正在 React / Vue / Next.js / Astro / Tailwind 等项目中使用，请额外说明目标技术栈，让 AI 按你的项目结构生成。
