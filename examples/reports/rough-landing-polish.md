# rough-landing Polish Dry Run

## Context

- Scope: `examples/test-cases/rough-landing/index.html`
- Polish mode: Standard Polish
- Existing visual direction: dark AI landing page
- Goal: reduce AI-template smell without redesigning the whole page

## Polish Diagnosis

- Main roughness sources: AI-template copy, excessive badges, loud background glow, inconsistent radius, and fixed-width hero elements.
- This should not become a new visual direction. It should remain a dark landing page, but quieter and more credible.

## Polish Summary

- 保留了什么：
  - Dark landing page direction.
  - Basic hero + mockup structure.
  - Product name and main action area.

- 调整了什么：
  - Reduce hero headline intensity and improve line-height/letter spacing.
  - Lower background glow strength.
  - Normalize radius across buttons, mockup, metrics, and cards.
  - Improve CTA wrapping and mobile behavior.

- 删除了什么：
  - Excessive badges.
  - Invented or suspicious metrics.
  - Decorative cards that do not explain product value.

## Before / After

- Before: page feels like a generic AI SaaS template with loud effects and vague claims.
- After: page should feel like a calmer, more credible dark product landing page.

## Should Not Change

- Do not switch to a light theme.
- Do not adopt cinematic-minimal unless explicitly requested.
- Do not invent new customer proof or metrics.
- Do not redesign into a dashboard.

## Recheck

- Mobile CTA and mockup no longer overflow.
- Badges and metrics no longer dominate the hero.
- Radius system is more consistent.
- Page still reads as the same product direction.
